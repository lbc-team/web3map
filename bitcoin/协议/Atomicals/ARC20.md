# ARC20

ARC20 是 Atomicals 协议下的可替代代币标准，旨在比特币区块链上实现去中心化、可互换的数字资产。与 Ordinals 协议下的 BRC-20 代币不同，ARC20 强调将代币余额直接绑定到比特币的 UTXO (未花费交易输出) 上，从而实现更原生的“原子性”和与比特币安全模型的紧密结合。

## 要解决的问题

1.  **缺乏原生资产标准**：比特币本身主要关注交易和 UTXO 管理，缺乏一种灵活且去中心化的原生代币发行和管理标准。
2.  **中心化风险**：现有的链下资产管理或依赖中心化索引器的代币方案，存在一定的中心化风险和数据不一致性。
3.  **互操作性挑战**：如何在比特币 UTXO 模型上，以一种简单、原子化的方式实现代币的铸造、转移和销毁，并易于被索引器理解和验证。

ARC20 旨在提供一个直接且去中心化的比特币代币标准，通过将代币余额与聪（Satoshi）紧密关联，最大化地继承比特币的安全性。

## 实现机制与原理

ARC20 代币的核心机制是**“聪着色”**（Sat Coloring）和**UTXO 原子绑定**。

### 代币铸造 (Minting)
1.  **Commit 阶段**：用户通过一笔特殊的比特币交易，在 `OP_RETURN` 输出中嵌入一个 JSON 对象。这个 JSON 对象包含了代币的元数据（如名称、符号、总量、精度）和一个 `mint` 操作指令。这笔交易的第一个输入 UTXO 被视为“初始原子”（initial atom）。
2.  **Reveal 阶段**：铸造者需要发送一笔交易，该交易花费“初始原子”所在的 UTXO，并将代币的供应量分配到新的 UTXO 中。每次铸造（mint）行为都要求用户提供比特币算力证明（PoW），这使得代币发行更公平且抗女巫攻击。

### 代币转移 (Transferring)
1.  ARC20 代币的余额直接与承载它的比特币 UTXO 绑定。例如，如果一个 UTXO 包含 1000 聪，并且该聪被“着色”为某种 ARC20 代币，那么这个 UTXO 就代表了 1000 个该 ARC20 代币。
2.  转移 ARC20 代币，即是**转移承载该代币的比特币 UTXO**。当一个比特币 UTXO 被花费并生成新的 UTXO 时，ARC20 代币的余额也会按照比特币交易的输入和输出规则进行分割和转移。
3.  协议规定，所有包含 ARC20 代币的 UTXO，其金额（聪的数量）必须等于代币余额。这意味着一个 UTXO 只能承载一种 ARC20 代币。

### 索引与验证
Atomicals 索引器负责扫描比特币区块链，解析包含 Atomicals 协议数据的 `OP_RETURN` 输出，并构建一个链下索引来追踪所有 ARC20 代币的铸造、转移和所有权状态。由于代币余额与 UTXO 紧密绑定，索引器可以相对容易地验证交易的有效性。

## 主要特点

*   **原子性**：ARC20 代币与比特币的聪直接绑定，其所有权和转移遵循比特币 UTXO 的原子性原则。这意味着代币的安全性与比特币区块链的安全性高度一致。
*   **公平铸造 (Proof-of-Work Minting)**：代币的铸造需要通过比特币的算力证明，这旨在促进更公平的代币分发，防止早期的大规模囤积。
*   **去中心化**：协议的设计不依赖于中心化的索引器来定义代币的有效性，虽然索引器提供了便利，但其数据可以通过协议规则进行验证。
*   **UTXO 绑定**：每个代币单元都与一个特定的 UTXO 关联，使资产管理更加明确和透明。
