## x402

### 概念简介

x402 是一个基于 HTTP 402 状态码的 Web3 原生支付协议，由 Base 链提出并推动。HTTP 402 "Payment Required" 状态码在传统互联网中长期被保留但很少使用，x402 协议将其重新定义为区块链支付的标准接口，使得 Web 应用可以无缝集成加密货币支付功能。

该协议旨在解决 Web2 到 Web3 支付的桥接问题，让开发者可以像处理传统 HTTP 状态码一样处理链上支付请求。通过标准化的 API 和响应格式，x402 降低了 Web3 支付集成的技术门槛，为内容付费、订阅服务、API 调用计费等场景提供了统一的解决方案。

作为 Base 生态的重要基础设施之一，x402 协议与智能钱包、账户抽象等技术结合，为用户提供流畅的支付体验，无需复杂的钱包操作或 Gas 管理。

### 核心特性

**HTTP 402 状态码标准化**
x402 协议将 HTTP 402 状态码重新定义为支付请求信号。当服务器返回 402 状态码时，响应头中包含支付详情（金额、接收地址、支持的代币等），客户端可以自动解析并完成支付。这种设计使得支付流程与现有 Web 基础设施无缝集成。

**多代币支持**
协议支持多种加密货币和代币支付，包括 ETH、稳定币（USDC、DAI）以及项目代币。开发者可以灵活配置接受的支付方式，用户可以选择自己持有的资产进行支付。

**账户抽象集成**
x402 深度集成 ERC-4337 账户抽象标准，支持智能合约钱包、社交恢复、Gas 代付等高级功能。用户无需手动管理私钥和 Gas 费用，支付体验接近传统互联网支付。

**即时确认**
通过 Base Layer2 的快速区块时间和低延迟特性，x402 支付可以在数秒内完成确认。对于小额支付，协议还支持乐观确认机制，进一步提升用户体验。

**开发者友好**
协议提供了完整的 SDK 和库，支持主流编程语言（JavaScript、Python、Go 等）。开发者只需几行代码即可为现有应用添加 x402 支付功能，无需深入了解区块链技术细节。

### 核心优势

**降低集成门槛**
通过复用熟悉的 HTTP 状态码机制，x402 让 Web2 开发者可以快速理解和集成 Web3 支付，无需学习全新的协议栈。

**跨平台兼容性**
协议设计遵循 Web 标准，可以在浏览器、移动应用、服务器端等各种环境中使用，实现真正的跨平台支付。

**可扩展性**
基于 Base Layer2 的高性能基础设施，x402 可以支持大规模并发支付请求，满足商业应用的需求。

**安全性保障**
支付信息通过智能合约验证，防止篡改和欺诈。用户资金始终在自己控制之下，避免了托管风险。

### 应用场景

**内容付费**：文章、视频、音频等数字内容的按次付费或订阅服务，创作者可以直接从用户获得收益。

**API 计费**：开发者 API、AI 模型调用、数据服务等按使用量计费，实现精细化的微支付。

**订阅服务**：SaaS 应用、会员服务、在线工具等的订阅付费，支持自动续费和灵活计费周期。

**游戏内购**：区块链游戏的道具购买、功能解锁，提供流畅的支付体验。

**小费和打赏**：社交平台、直播应用中的创作者打赏功能，降低支付摩擦。
